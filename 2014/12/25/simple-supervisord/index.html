supervisorctl是supervisor的命令行控制台. 可以查看控制进程的状态.

RUNNING->STARTING->RUNNING表示正在重启. 重启后pid更新.

异常处理

如果出现异常情况, 可以去配置项: logfile=/var/log/supervisor/supervisord.log 配置的log中查看详情.

疑难问题

  1. 被监控进程, 必须由supervisord来启动.
  2. 被监控进程必须不是daemon程序.
  3. Another program is already listening on a port that one of our HTTP serve rs is configured to use. Shut this program down first before starting superviso rd. 如果supervisor已经启动, 只需要kill掉. 否则, 参照解决方案
