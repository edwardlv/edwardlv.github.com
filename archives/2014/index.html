分模块解析

如何登录

  1. 首先明确captcha和recaptcha的区别. 这两者都比较损伤用户体验, 推荐作为保留手段. 对比captcha, recaptcha对于机器更难识别, 相对好点.
  2. 浏览器自动提示功能虽然可以关闭. 但是用户不同网站有很多账户, 可能会忘记那么多账户&密码.
  3. 对于网络嗅探, 一定使用基于加密(ssl) 或者 proven & tested challenge-response scheme (for example, the Diffie-Hellman-based SRP) 的方式. 其他任何其他方式都会被黑客中间窃听. 仅仅基于客户端的加密方式也是无效的, 除非结合上述两种方法. 哈希密码, 可以有效防止密码明文泄露, 但是无法防止重放攻击, 中间人攻击/劫持, 暴力破解(username, salt, hashed password).
  4. 发送完认证token后, 服务器端必须存储session记录用户已经被授权. cookie可以存储用户session的索引. Cookie必须有secure flag 和 Http only flag. httponly flag防止cookie被作为xss攻击. secure flag可以保证cookie仅通过 https 发送回来. cookie必须不可预知. 当cookie指向一个不存在的session时, cookie的值需要立刻被重置, 防止 session fixation.

xmpp协议介绍

什么是XMPP?

xmpp(The eXtensible Messaging and Presence Protocol, aka as Jabber)一种IM通讯数据交换协议, 数据格式为XML. 该数据交换协议定义了3种角色: client, server, gate, 两两之间可以基于xmpp协议相互通信. xmpp server是基于域名服务, client@jabber.server_A 也可以将信息发送到 jabber.server_B. 此外, 通过xmpp gate甚至可以发送消息到非xmpp server.

什么是JID

JID的格式, node@server/resource.

  • node表示某个server上唯一一个人.
  • server代表了node所在的server.
  • resource代表相同用户的不同场景. (实际区别不大?)

node@server/resource 三者决定一个唯一的jid.

信息流

Jabber信息是xml格式, 基本交流模式 client<-->server, 比如: UserA 和 UserB 聊天, 信息流:

http://ceit.uq.edu.au/content/how-xmpp-works-step-step总结了client和server交流模式. http://blog.csdn.net/huyoo/article/details/24353105做了翻译.

个人总结: 一次 client & server 沟通, client和server 均以 <stream>开始, 服务器端开始维护一个session, </stream>表示结束, 回收session, 关闭连接.
